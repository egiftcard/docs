Allow users to connect to your app by generating a [Local Wallet](/glossary/local-wallet) directly in your application.

A local wallet is a low-level wallet that allows you to create wallets within your application or project. It is a non-custodial solution that simplifies the onboarding process by offering a "checkout as guest" experience. This improves the user experience for web3 apps in two ways:

1. It enables non-web3 native users to get started easily without having to create a wallet with an external provider like MetaMask. 
2. It hides transaction confirmations from users. With a local wallet, you can generate wallets on the backend. These wallets are app-scoped meaning that each user has one wallet per app. 

Additionally, the local wallet has UI functionality, available within the [React](/react) and [React Native](/react-native) SDKs, that allows users to export and optionally store the encrypted key for the local wallet on their browser, mobile, or other devices.

## Usage

```javascript
import { LocalWallet } from "@thirdweb-dev/wallets";

const wallet = new LocalWallet();

// generate a random wallet
await wallet.generate();
// connect the wallet to the application
await wallet.connect()
```

## Configuration

Optionally, provide a configuration object when instantiating the `LocalWallet` class.

<details>

```javascript
import { LocalWallet } from "@thirdweb-dev/wallets";
import { Goerli } from "@thirdweb-dev/chains";

const walletWithOptions = new LocalWallet(
  // highlight-start

    chain: Goerli,
  },
  // highlight-end
);
```

</details>

## Methods

Once instantiated, utilize the following methods to interact with the Local Wallet.

### addListener

Add a listener for a specific event, and run a callback whenever that event is emitted.

```javascript
wallet.addListener("connect", (data) => console.log(data));
```

<details>
  <summary>Configuration</summary>
  <div>

#### eventName

The name of the event to listen for.
Each event returns a different `data` object, or no `data` object at all.

Possible options:

- [`change`](#change)
- [`connect`](#connect)
- [`disconnect`](#disconnect)
- [`error`](#error)
- [`message`](#message)
- [`open_wallet`](#open_wallet)
- [`request`](#request)

##### change

User changes the wallet they are connected with, or the chain they are connected to.

Data available:

```typescript
{
  address?: string; // available if the address is changed
  chainId?: number; // available if the network is changed
}
```

##### connect

User connects their wallet to your app.

Data available:

```typescript
{
  address: string;
  chainId: number;
}
```

##### disconnect

User disconnects their wallet from your app.

No `data` is available.

##### error

An error occurs in the wallet.

Data available:

```typescript
{
  cause: unknown;
  message: string;
  name: string;
  stack: string | undefined;
}
```

##### message

Wallet receives some message that the consumer should be notified of. The kind of message is identified by the `type` field.

Data available:

```typescript
{
  type: string;
  data: unknown;
}
```

##### open_wallet

User opens the wallet app.

`data` is a `string | undefined`.

##### request

Wallet receives some request that the consumer should be notified of. The kind of request is identified by the `type` field.

No `data` is available.

#### callback

The function to run when the event is emitted.

</div>
</details>

### emit

```javascript

```

<details>
  <summary>Configuration</summary>

#### data

Optional data to send with the event. Must be an object with information relevant to the event.

See [addListener](#addlistener) configuration for more information on the different events.

### export

```javascript

    strategy: "encryptedJson",
    password: "password",
});
```

```typescript
    strategy: "privateKey";

} | {
    strategy: "mnemonic";
    encryption: EncryptOptions;
};
```

Example:

```javascript
wallet.export({
    // highlight-start

    password: "password",
    // highlight-end
```

#### Return Value

```typescript
Promise<string>;
```
