Let your users connect to their Smart Wallet.

A Smart Wallet is a wallet that is controlled by a smart contract following the [ERC-4337 specification](https://eips.ethereum.org/EIPS/eip-4337).

## Features

Smart Wallets work a little differently than other wallets since it's controlled by a programmable smart contract rather than a regular EOA. This programmable wallet contract can:

- Holds any assets
- Pay for its own gas
- Execute transactions in batches
- Have multiple owners
- Assign roles and permissions

And more! Smart Wallets can be customized to each application's needs using the Solidity SDK.

## Requirements

- A deployed account factory contract, you can deploy one in 1-click via the [explore page](https://thirdweb.com/explore)
- An API key, which you can obtain from the [thirdweb dashboard](https://thirdweb.com/dashboard)

The API key let's you access thirdweb's bundler and paymaster infrastructure, which is required for the smart wallet to operate and optionally enable gasless transaction.

The SmartWallet API is composable - you can bring in your own 4337-compatible factory contract and you can also use your own bundler or paymaster services.

## Usage

**To connect to a smart wallet, a personal wallet (acting as the key to the smart wallet) must first be connected.**

```javascript
import { LocalWallet, SmartWallet } from "@thirdweb-dev/wallets";
import { Goerli } from "@thirdweb-dev/chains";

// First, connect the personal wallet, can be any wallet (metamask, walletconnect, etc.)
const personalWallet = new LocalWallet();
await personalWallet.connect();

// Setup the Smart wallet configuration
const config: SmartWalletConfig = {
  chain: Goerli, // the chain where your smart wallet will be or is deployed
  factoryAddress: "{{factory_address}}", // your own deployed account factory address
  thirdwebApiKey: "{{api_key}}", // obtained from the thirdweb dashboard
  gasless: true, // enable or disable gasless transactions
};

// Then, connect the Smart wallet
const wallet = new SmartWallet(config);
await wallet.connect({
  personalWallet,
});
```

## Configuration

Here's the full configuration object you can pass when instantiating the `SmartWallet` class.

### Mandatory properties

<details>
  <summary>chain</summary>
  <div>

The chain that the Smart Wallet contract is deployed to.

Either a `Chain` object, from the [`@thirdweb-dev/chains`](https://www.npmjs.com/package/@thirdweb-dev/chains) package, or a chain name, or a RPC url.

</div>
</details>

<details>
  <summary>factoryAddress</summary>
  <div>

The address of the Smart Wallet Factory contract.

Must be a `string`.

</div>
</details>

<details>
  <summary>thirdwebApiKey</summary>
  <div>

Your thirdweb API key, can be obtained from the [thirdweb dashboard](https://thirdweb.com/dashboard).

If you're using your own bundler and paymaster, you can set this to empty string.

Must be a `string`.

</div>
</details>

<details>
  <summary>gasless</summary>
  <div>

Whether to turn on or off gasless transactions.

- If set to `true`, all gas fees will be paid by a paymaster.
- If set to `false`, all gas fees will be paid by the Smart Wallet itself (needs to be funded).

Must be a `boolean`.

</div>
</details>

### Optional properties

<details>
  <summary>factoryInfo</summary>
  <div>

Customize how the Smart Wallet Factory contract is interacted with. If not provided, the default functions will be used.

Must be a `object`. The object can contain the following properties:

- `createAccount` - a function that returns the transaction object to create a new Smart Wallet.
- `getAccountAddress` - a function that returns the address of the Smart Wallet contract given the owner address.
- `abi` - optional ABI. If not provided, the ABI will be auto resolved.

Example:

```javascript
 const config: SmartWalletConfig = {
      chain,
      gasless,
      factoryAddress,
      thirdwebApiKey,
      // highlight-start
      factoryInfo: {
        createAccount: (factory, owner) => {
          return factory.prepare("customCreateAccount", [
            owner,
            getExtraData(),
          ]);
        },
        getAccountAddress(factory, owner) {
          return factory.call("getAccountAddress", [owner]);
        },
        abi: [...]
      },
      // highlight-end
    };
```

</div>
</details>

<details>
  <summary>accountInfo</summary>
  <div>

Customize how the Smart Wallet Account contract is interacted with. If not provided, the default functions will be used.

Must be a `object`. The object can contain the following properties:

- `execute` - a function that returns the transaction object to execute an arbitrary transaction.
- `getNonce` - a function that returns the current nonce of the account.
- `abi` - optional ABI. If not provided, the ABI will be auto resolved.

Example:

```javascript
 const config: SmartWalletConfig = {
      chain,
      gasless,
      factoryAddress,
      thirdwebApiKey,
      // highlight-start
      accountInfo: {
        execute: (account, target, value, data) => {
          return account.prepare("customExecute", [
            target, value, data
          ]);
        },
        getNonce(account) {
          return account.call("getNonce");
        },
        abi: [...]
      },
      // highlight-end
    };
```

</div>
</details>

<details>
  <summary>bundlerUrl</summary>
  <div>

Your own bundler URL to send user operations to. Uses thirdweb's bundler by default.

Must be a `string`.

</div>
</details>

<details>
  <summary>paymasterUrl</summary>
  <div>

Your own paymaster URL to send user operations to for gasless transactions. Uses thirdweb's paymaster by default.

Must be a `string`.

</div>
</details>

<details>
  <summary>entryPointAddress</summary>
  <div>

The entrypoint contract address. Uses v0.6 by default.

Must be a `string`.

</div>
</details>

<details>
  <summary>chains</summary>
  <div>

Provide an array of chains you want to support.

Must be an array of `Chain` objects, from the [`@thirdweb-dev/chains`](https://www.npmjs.com/package/@thirdweb-dev/chains) package.

Defaults to our [default chains](/react/react.thirdwebprovider#default-chains).

```javascript
import { SafeWallet } from "@thirdweb-dev/wallets";
import { Ethereum } from "@thirdweb-dev/chains";

const walletWithOptions = new SafeWallet(
  // highlight-start
  {
    chains: [Ethereum],
  },
  // highlight-end
);
```

</div>
</details>

<details>
  <summary>dappMetadata</summary>
  <div>

Information about your app that the wallet will display when your app tries to connect to it.

Must be an object containing `name`, `url`, `description` and `logoUrl` properties.

```javascript
import { SafeWallet } from "@thirdweb-dev/wallets";

const walletWithOptions = new SafeWallet({
  // highlight-start
  dappMetadata: {
    name: "thirdweb powered dApp",
    url: "https://thirdweb.com",
    description: "thirdweb powered dApp",
    logoUrl: "https://thirdweb.com/favicon.ico",
  },
  // highlight-end
});
```

</div>
</details>

<details>
  <summary>walletId</summary>
  <div>

An ID for the wallet used to store the wallet in the `walletStorage`.

```javascript
import { SafeWallet } from "@thirdweb-dev/wallets";

const walletWithOptions = new SafeWallet(
  // highlight-start
  {
    walletId: "wallet-id",
  },
  // highlight-end
);
```

</div>
</details>

## Methods

Once instantiated, utilize the following methods to interact with the wallet.

### addListener

Add a listener for a specific event, and run a callback whenever that event is emitted.

```javascript
wallet.addListener("connect", (data) => console.log(data));
```

<details>
  <summary>Configuration</summary>
  <div>

#### eventName

The name of the event to listen for.
Each event returns a different `data` object, or no `data` object at all.

Possible options:

- [`change`](#change)
- [`connect`](#connect)
- [`disconnect`](#disconnect)
- [`error`](#error)
- [`message`](#message)
- [`open_wallet`](#open_wallet)
- [`request`](#request)

##### change

User changes the wallet they are connected with, or the chain they are connected to.

Data available:

```typescript
{
  address: string;
  chainId: number;
}
```

##### connect

User connects their wallet to your app.

Data available:

```typescript
{
  address: string;
  chainId: number;
}
```

##### disconnect

User disconnects their wallet from your app.

No `data` is available.

##### error

An error occurs in the wallet.

Data available:

```typescript
{
  cause: unknown;
  message: string;
  name: string;
  stack: string | undefined;
}
```

##### message

Wallet receives some message that the consumer should be notified of. The kind of message is identified by the `type` field.

Data available:

```typescript
{
  type: string;
  data: unknown;
}
```

##### open_wallet

User opens the wallet app.

`data` is a `string | undefined`.

##### request

Wallet receives some request that the consumer should be notified of. The kind of request is identified by the `type` field.

No `data` is available.

#### callback

The function to run when the event is emitted.

</div>
</details>

### autoConnect

Attempts to connect to the wallet without asking the user&rsquo; explicitly;
possible if the user has already connected to the wallet to your app before.

```javascript
const address = await wallet.autoConnect();
```

<details>
  <summary>Configuration</summary>
  <div>

#### Return Value

Returns a `string` containing the wallet address, or `undefined` if the connection failed.

```typescript
string | undefined;
```

</div>
</details>

### connect

Prompt the user to connect their smart wallet to your app.

Note: A personal wallet must be connected before connecting to a Smart wallet.

```javascript
import { CoinbaseWallet, SmartWallet } from "@thirdweb-dev/wallets";
import { Ethereum } from "@thirdweb-dev/chains";

// First, connect the personal wallet
const personalWallet = new CoinbaseWallet();
await personalWallet.connect();

// Then, connect the Smart wallet
const wallet = new SmartWallet(config);
await wallet.connect({
  personalWallet,
});
```

<details>
  <summary>Configuration</summary>
  <div>

#### personalWallet

The instance of a personal wallet that can sign transactions on the Smart Wallet.

Must be an `EVMWallet` instance such as [`CoinbaseWallet`](/wallet/coinbase-wallet) or [`MetamaskWallet`](/wallet/metamask).

Returns a `string` containing the wallet address, or `undefined` if the connection failed.

```typescript
string | undefined;
```

</div>
</details>

### disconnect

Disconnect the currently connected wallet from your app.

```javascript
await wallet.disconnect();
```

### emit

Emit an event, with some optional data.

```javascript
wallet.emit("connect", {});
```

<details>
  <summary>Configuration</summary>
  <div>

#### eventName

The name of the event to emit.

Must be one of: `change`, `connect`, `disconnect`, `error`, `message`, `open_wallet`, `request`.

#### data

Optional data to send with the event. Must be an object with information relevant to the event.

See [addListener](#addlistener) configuration for more information on the different events.

</div>
</details>

### eventNames

Get an array of the events for which the emitter has registered listeners.

```javascript
const eventNames = wallet.eventNames();
```

<details>
  <summary>Configuration</summary>
  <div>

#### Return Value

Returns an array of `string`s containing the event names.

```typescript
(`change` | `connect` | `disconnect` | `error` | `message` | `open_wallet` | `request`)[];
```

</div>
</details>

```typescript
string | undefined;
```

</div>
</details>

```javascript
const chainId = await wallet.getChainId();
```

```typescript
number | undefined;
```

</div>
</details>

```javascript
const metadata = wallet.getMeta();
```

```javascript
const connectedPersonalWallet = wallet.getPersonalWallet();
```

</details>

```typescript
string | null;
```

</div>
</details>

```typescript
Signer;
```

</div>
</details>

### listenerCount

Get the number of listeners for a given event.

```javascript
const listenerCount = wallet.listenerCount("connect");
```

<details>
  <summary>Configuration</summary>
  <div>

#### eventName

The name of the event to get the listener count for.

Must be one of: `change`, `connect`, `disconnect`, `error`, `message`, `open_wallet`, `request`.

#### Return Value

Returns a `number` containing the number of listeners for the event.

```typescript
number;
```

</div>
</details>

### listeners

Get an array of the listeners for a given event.

```javascript
const listeners = wallet.listeners("connect");
```

<details>
  <summary>Configuration</summary>
  <div>

#### eventName

The name of the event to get the listeners for.

Must be one of: `change`, `connect`, `disconnect`, `error`, `message`, `open_wallet`, `request`.

#### Return Value

Returns an array of `function`s containing the listeners for the event.

```typescript
((data: WalletData) => void)[]
```

</div>
</details>

### removeAllListeners

Remove all event listeners for either all events, or a specific event.

```javascript
wallet.removeAllListeners();
```

<details>
  <summary>Configuration</summary>
  <div>

#### eventName

Optionally, provide the name of the event to remove all listeners for.

Must be one of: `change`, `connect`, `disconnect`, `error`, `message`, `open_wallet`, `request`.

```typescript
wallet.removeAllListeners("connect");
```

</div>
</details>

### removeListener

Remove a specific event listener for a given event.

Provide the event name and the listener function to remove.

```javascript
wallet.removeListener("connect", (data) => console.log(data));
```

<details>
  <summary>Configuration</summary>
  <div>

#### eventName

The name of the event to remove the listener for.

Must be one of: `change`, `connect`, `disconnect`, `error`, `message`, `open_wallet`, `request`.

#### listener

The listener function to remove. i.e. provide the same function that was used to add the listener.

</div>
</details>

### signMessage

Sign a message with the currently connected wallet.

```javascript
const signature = await wallet.signMessage("Hello world!");
```

<details>
  <summary>Configuration</summary>
  <div>

#### message

The message to sign.

Must be a `string`.

#### Return Value

Returns a `string` containing the signature.

```typescript
string;
```

</div>
</details>

### switchChain

Switch the currently connected wallet to a different chain.

```javascript
await wallet.switchChain(1);
```

<details>
  <summary>Configuration</summary>
  <div>

#### chainId

The chain ID to switch to.

Must be a `number`.

</div>
</details>

### verifySignature

Verify a signature is valid.

```javascript
const isValid = await wallet.verifySignature(
  "Message",
  "Signature",
  "wallet_address",
);
```

<details>
  <summary>Configuration</summary>
  <div>

#### message

The message that was signed.

Must be a `string`.

#### signature

The signature to verify.

Must be a `string`.

#### address

The address of the wallet that signed the message.

Must be a `string`.

#### chainId

The chain ID to check the signature on.

Optional, must be a `number`.

</div>
</details>
