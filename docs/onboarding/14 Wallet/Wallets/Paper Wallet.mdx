Prompt users to connect to your app using their email with [Paper Wallet](https://docs.withpaper.com/reference/embedded-wallet-service-overview)

## Usage

```javascript
import { PaperWallet } from "@thirdweb-dev/wallets";
import { Ethereum } from "@thirdweb-dev/chains";

const wallet = new PaperWallet({
  chain: Ethereum, //  chain to connect to
  clientId: "client_id", // Paper SDK client ID
});

wallet.connect();
```

## Configuration

Provide a configuration object when instantiating the `PaperWallet` class.

<details>
  <summary>chain (required)</summary>
  <div>

The chain to connect to by default.

Must be a `Chain` object, from the [`@thirdweb-dev/chains`](https://www.npmjs.com/package/@thirdweb-dev/chains) package.

</div>
</details>

<details>
  <summary>chains (optional)</summary>
  <div>

Provide an array of chains you want to support.

Must be an array of `Chain` objects, from the [`@thirdweb-dev/chains`](https://www.npmjs.com/package/@thirdweb-dev/chains) package.

</div>
</details>

<details>
  <summary>clientId (required)</summary>
  <div>

Paper SDK requires a clientId for instantiation. You can create a clientId for your app on [paper.xyz](https://paper.xyz/product/wallet)

Must be a `string`.

</div>
</details>

<details>
  <summary>walletStorage (optional)</summary>
  <div>

wallet needs to store data in a persistent storage. By default `localStorage` is used. If you want to use a different storage, you can pass it in the `walletStorage` option.

Must be an object conforming to the `AsyncStorage` interface:

```typescript
export interface AsyncStorage {
  getItem(key: string): Promise<string | null>;
  setItem(key: string, value: string): Promise<void>;
  removeItem(key: string): Promise<void>;
}
```

Example:

```javascript
import { PaperWallet } from "@thirdweb-dev/wallets";

const walletWithOptions = new PaperWallet({
  // ... other required config

  // highlight-start
  walletStorage: {
    getItem: (key) => {
      // Implement your own storage logic here
    },
    removeItem: (key) => {
      // Implement your own storage logic here
    },
    setItem: (key, value) => {
      // Implement your own storage logic here
    },
  },
  // highlight-end
});
```

</div>
</details>

## Methods

Once instantiated, utilize the following methods to interact with the Paper Wallet.

### addListener

Add a listener for a specific event, and run a callback whenever that event is emitted.

```javascript
wallet.addListener(eventName, (data) => console.log(data));
```

<details>
  <summary>Configuration</summary>
  <div>

#### eventName

The name of the event to listen for.
Each event returns a different `data` object, or no `data` object at all.

Possible options:

- [`change`](#change)
- [`connect`](#connect)
- [`disconnect`](#disconnect)
- [`error`](#error)
- [`message`](#message)
- [`open_wallet`](#open_wallet)
- [`request`](#request)

##### change

User changes the wallet they are connected with, or the chain they are connected to.

Data available:

```typescript
{
  address: string;
  chainId: number;
}
```

##### connect

User connects their wallet to your app.

Data available:

```typescript
{
  address: string;
  chainId: number;
}
```

##### disconnect

User disconnects their wallet from your app.

No `data` is available.

##### error

An error occurs in the wallet.

Data available:

```typescript
{
  cause: unknown;
  message: string;
  name: string;
  stack: string | undefined;
}
```

##### message

Wallet receives some message that the consumer should be notified of. The kind of message is identified by the `type` field.

Data available:

```typescript
{
  type: string;
  data: unknown;
}
```

##### open_wallet

User opens the wallet app.

`data` is a `string | undefined`.

##### request

Wallet receives some request that the consumer should be notified of. The kind of request is identified by the `type` field.

No `data` is available.

#### callback

The function to run when the event is emitted.

</div>
</details>

### autoConnect

Attempts to connect to the wallet without asking the user explicitly.

This only works if the user connected the wallet to your app before.

```javascript
const address = await wallet.autoConnect();
```

<details>
  <summary>Configuration</summary>
  <div>

#### Return Value

Returns a `string` containing the wallet address, or throws an error if autoconnect fails

```typescript
string;
```

</div>
</details>

### connect

Opens a Modal and prompts the user to login using their email.

Once user enters their email, an OTP is sent to their email. User can then enter the OTP in the modal. Once that is done, the wallet is connected to the app.

```javascript
const address = await wallet.connect();
```

If you know ther user's email already, you can pass the `email` to `connect` method and the modal will directly show the UI to enter the OTP sent to user's email.

```javascript
const address = await wallet.connect({
  email: "user@example.com",
});
```

<details>
  <summary>Configuration</summary>
  <div>

#### Return Value

Returns a `string` containing the wallet address, or throws an error if connection fails

```typescript
string;
```

</div>
</details>

### disconnect

Disconnect the wallet from your app and logout from paper. Autoconnect won't work once user is logged out.

```javascript
await wallet.disconnect();
```

### emit

Emit an event, with some optional data.

```javascript
wallet.emit(eventName, data);
```

<details>
  <summary>Configuration</summary>
  <div>

#### eventName

The name of the event to emit.

Must be one of: `change`, `connect`, `disconnect`, `error`, `message`, `open_wallet`, `request`.

#### data

Optional data to send with the event. Must be an object with information relevant to the event.

See [addListener](#addlistener) configuration for more information on the different events.

</div>
</details>

### eventNames

Get an array of the events for which the emitter has registered listeners.

```javascript
const eventNames = wallet.eventNames();
```

<details>
  <summary>Configuration</summary>
  <div>

#### Return Value

Returns an array of `string`s containing the event names.

```typescript
(`change` | `connect` | `disconnect` | `error` | `message` | `open_wallet` | `request`)[];
```

</div>
</details>

### listenerCount

Get the number of listeners for a given event.

```javascript
const listenerCount = wallet.listenerCount(eventName);
```

<details>
  <summary>Configuration</summary>
  <div>

#### eventName

The name of the event to get the listener count for.

Must be one of: `change`, `connect`, `disconnect`, `error`, `message`, `open_wallet`, `request`.

#### Return Value

Returns a `number` containing the number of listeners for the event.

```typescript
number;
```

</div>
</details>

### listeners

Get an array of the listeners for a given event.

```javascript
const listeners = wallet.listeners(eventName);
```

<details>
  <summary>Configuration</summary>
  <div>

#### eventName

The name of the event to get the listeners for.

Must be one of: `change`, `connect`, `disconnect`, `error`, `message`, `open_wallet`, `request`.

#### Return Value

Returns an array of `function`s containing the listeners for the event.

```typescript
((data: WalletData) => void)[]
```

</div>
</details>

### removeAllListeners

Remove all event listeners for either all events, or a specific event.

```javascript
wallet.removeAllListeners(); // all events
wallet.removeAllListeners(eventName); // specific event
```

<details>
  <summary>Configuration</summary>
  <div>

#### eventName

Optionally, provide the name of the event to remove all listeners for.

Must be one of: `change`, `connect`, `disconnect`, `error`, `message`, `open_wallet`, `request`.

```typescript
wallet.removeAllListeners("connect");
```

</div>
</details>

### removeListener

Remove a specific event listener for a given event.

Provide the event name and the listener function to remove.

```javascript
wallet.removeListener(eventName, callback);
```

<details>
  <summary>Configuration</summary>
  <div>

#### eventName

The name of the event to remove the listener for.

Must be one of: `change`, `connect`, `disconnect`, `error`, `message`, `open_wallet`, `request`.

#### listener

The listener function to remove. i.e. provide the same function that was used to add the listener.

</div>
</details>

#### Return Value

</div>
</details>

### switchChain

```javascript
await wallet.switchChain(chainId);
```

<details>
  <summary>Configuration</summary>
  <div>

#### chainId

The chain ID to switch to.

Must be a `number`.

</div>
</details>

### verifySignature

Verify a signature is valid.

```javascript
const isValid = await wallet.verifySignature(message, signature, address);
```

<details>
  <summary>Configuration</summary>
  <div>

#### message

The message that was signed.

Must be a `string`.

#### signature

The signature to verify.

Must be a `string`.

#### address

The address of the wallet that signed the message.

Must be a `string`.

#### chainId

The chain ID to check the signature on.

Optional, must be a `number`.

</div>
</details>
