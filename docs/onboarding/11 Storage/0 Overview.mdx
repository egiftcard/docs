# Storage

Storage is a decentralized file management solution to store and retrieve information "off-chain" using [IPFS](/glossary/ipfs). It enables you to
upload and [pin](https://docs.ipfs.tech/concepts/persistence/) files to IPFS, and retrieve them at any time via our [IPFS gateway](#ipfs-gateway).

Using either the [dashboard](https://thirdweb.com/dashboard/storage), [CLI](/cli), or [SDK](/sdk), you can easily upload and pin individual files, or entire directories to IPFS;
ideal for storing files in a permanent, decentralized, censorship-resistant manner.

Storage powers functionality in a number of our other products, such as uploading smart contract ABIs in [deploy](/deploy),
our [IPFS Renderer](/react/react.mediarenderer) UI component & [NFT metadata fetching](/react/react.usenft) capabilities in the SDK, and uploading of NFT metadata in the dashboard.

## IPFS Gateway

To read data from IPFS, an [IPFS Gateway](https://docs.ipfs.tech/concepts/ipfs-gateway/) is required; which
allows you to access data from the IPFS protocol on browsers and other HTTP clients, such as when building an application using our [SDK](/sdk).

Out of the box, we provide you with a public, unrestricted IPFS gateway at `https://ipfs.thirdwebcdn.com/ipfs/[cid]`,
which you can use to access any file available on IPFS for free; with no rate limits or restrictions.
